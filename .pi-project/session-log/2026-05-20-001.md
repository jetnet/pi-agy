## Architectural notes

- **agy 1.0.0 has only 8 flags**, and several documented ones (`-m`/`--model`, `--output-format`) are silently consumed as positional args. Model selection is TUI-only via `/model`; print mode (`-p`) inherits the last interactively-selected model. This forces M0-b (hardcode Flash 3.5).
- **Pi's extension auto-discovery has 3 paths**, not the 2 documented in `extensions.md`: in addition to `extensions/*.ts` and `extensions/*/index.ts`, the loader also reads each subdir's `package.json` for the `pi.extensions` field. pi-agy uses this third path (`"pi": { "extensions": ["./src/index.ts"] }`) so it loads on next pi startup without `settings.json` changes.
- **OAuth credentials live at `~/.gemini/oauth_creds.json` + `~/.gemini/google_accounts.json`** — both copied as bytes into profile dirs by `agy_account`. No `usage.json` exists on disk; agy's `/usage` comes from a live Google API call (which is why M2 is hard).
- **Renderer-result contract:** all 5 tools return `{ content: [{ type: "text", text }], details: AgyToolDetails, isError }`. `details` carries `{ durationMs, account, exitCode, model? }` and feeds the dimmed header line in `render.ts` (collapsed/expanded/error paths all use optional chaining so missing details degrades gracefully).
- **Subprocess invariant:** `spawnAgy()` is the only place that touches `child_process.spawn()`. It uses `shell: false` + argv array (no injection surface), buffers stdout/stderr into strings, kills via SIGTERM→SIGKILL with a stored timer ID, and cleans up the AbortSignal listener in `proc.on("close")` to prevent leaks across many calls (P1.5 fix).
- **Tool registration order** in `src/index.ts`: design / critique / image_to_ui / usage / account. Each tool gets its own `promptSnippet`, `promptGuidelines`, and shared `renderCall`/`renderResult` from `render.ts`. The promptSnippet/guidelines are the discoverability surface — they're what Opus reads to know when to invoke the tool. M1 will refine these based on observed routing behavior.
